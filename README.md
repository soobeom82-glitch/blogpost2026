## 이 프로젝트가 다루는 것

이 프로젝트는 "사업을 잘 포장해서 보여주는 블로그"가 아니라, 실제 운영에서 겪은 문제와 판단 과정을 기록하는 아카이브다.

핵심은 다음 두 가지다.

- 현실에서 발생한 사건을 남긴다.
- 그 사건이 비용, 운영, 판단, 수익에 어떤 영향을 줬는지 같이 남긴다.

## 핵심 원칙

1. AI 양산 콘텐츠 금지
2. 실제 경험 기반 작성
3. LLM은 초안 작성과 구조화 도구로만 활용
4. 모든 글에 본인 경험과 생각을 반드시 추가
5. 사진, 비용, 시행착오, 민원, 대응 과정을 적극 활용

## 기본 운영 방식

- 한 글에 한 사건만 잡는다.
- 사건 -> 당시 상황 -> 실제 대응 -> 결과 -> 다음에 다시 하면 어떻게 할지 순서로 쓴다.
- 가능하면 숫자를 넣는다.
- 가능하면 사진, 문자, 영수증, 메모를 같이 남긴다.
- 결론을 너무 교훈적으로 포장하지 않는다.

## 제작 방식

이 프로젝트의 글은 기본적으로 인터뷰 기반으로 만든다.

- 소재 선정은 Codex가 한다.
- 독자가 궁금해할 지점을 기준으로 Codex가 질문한다.
- 답변은 실제 경험 위주로 짧고 구체적으로 받는다.
- 초안 작성과 구조화는 Codex가 맡는다.
- 최종 글은 "내가 나를 인터뷰한 기록" 같은 형식으로 쓴다.

한 소재가 길면 한 편으로 억지로 압축하지 않는다.

- 사건 소개편
- 판단 미스편
- 운영 현실편
- 비용/수익 정리편

처럼 여러 편으로 나눠 연재한다.

## 우선순위

지금 부족한 것은 소재가 아니라 발행 시스템이다.

1. Vercel 배포 연결
2. 기본 카테고리 구성
3. 초반 30개 글 백로그 확보
4. 주간 발행 루틴 고정
